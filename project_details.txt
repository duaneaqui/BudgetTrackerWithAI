Create a full-stack web app called BudgetAI PH.

Goal:
Build a free, portfolio-ready personal finance tracker for Filipino users. The app should track income, expenses, recurring bills, loans, savings goals, local area prices, and provide AI-style budgeting insights. Do not implement paid bank APIs, GCash APIs, Maya APIs, BPI APIs, or real auto-syncing. This project must work with manual input and free OCR-based screenshot upload only.

Tech Stack:
Frontend:
- React
- TypeScript
- Tailwind CSS
- Recharts
- Vite or Next.js

Backend:
- Python
- FastAPI
- SQLite
- SQLAlchemy
- Pydantic
- JWT authentication if possible

OCR:
- Use Tesseract OCR or EasyOCR.
- If OCR setup is difficult, create a mock OCR parser first and structure the code so real OCR can be plugged in later.

10. Loan Screenshot Scanner
Create a feature where the user can upload a screenshot of a loan page, loan receipt, or loan message.

Frontend:
- Upload image button
- Preview uploaded image
- Button: “Scan Loan”
- Display extracted results in an editable confirmation form
- Button: “Save Loan”

Backend:
Endpoint:
POST /ocr/loan-screenshot

Process:
1. Receive uploaded image
2. Run OCR using Tesseract/EasyOCR or a mock parser
3. Extract possible loan fields:
   - provider
   - amount_due
   - remaining_balance
   - original_amount
   - monthly_payment
   - due_date
   - interest_or_fees
   - status
4. Return extracted data
5. User must confirm before saving

Important:
Never auto-save OCR results. Always require confirmation.

Example OCR text:
SLoan
Amount Due: ₱1,250
Due Date: June 15, 2026
Remaining Balance: ₱3,750

Expected parsed result:
{
  "provider": "SLoan",
  "monthly_payment": 1250,
  "due_date": "2026-06-15",
  "remaining_balance": 3750,
  "status": "active"
}
